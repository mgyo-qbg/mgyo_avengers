## 서비스 목적

재무 어드바이저(FA)가 고객의 자산/은퇴/투자를 종합적으로 설계·관리할 수 있는 B2B SaaS 플랫폼.

- FA가 데이터 기반 전문 재무 상담 제공
- 복잡한 금융 계산 자동화 (은퇴설계, 투자진단, 자산배분, 현금흐름)
- PDF 리포트 생성으로 대면 상담 자료 제공
- 마이데이터 연동으로 실제 금융 데이터 기반 분석

## 사용자 역할 (3가지)

| 역할 | 설명 |
|------|------|
| FA_ADMIN | 조직 관리자 — FA 생성/관리, 포트폴리오 관리, 디자인 관리 |
| ROLE_ADVISOR | 일반 재무 어드바이저 — 고객 관리, 투자/은퇴 설계 |
| ROLE_ABROAD_GBI | 자녀 유학 자금 전용 — 제한된 접근 |

### 외부/PDF

- `/external/` — 본인인증, 투자성향진단, 약관
- `/pdf/` — 20개 이상 세부 페이지 (표지/목차/본문/컴플라이언스 등)

### 백엔드 구조

- **be-nest**: famain(REST API) + fabatch(배치). MySQL(Prisma) + MongoDB + Redis
- **be-fastapi**: Nudge 마이크로서비스 — 재무 계산 엔진
- **외부 연동**: 마이데이터 기관, BetterApp, 카카오, NCP 메일, 단축URL

## 기획 이슈

### 비활성/미완성 기능
- 실적관리, 디자인관리, 알림, 수수료진단, 마켓컴패스, 투자비교, 사용량 계산기
- 이 기능들의 로드맵(활성/비활성/폐기) 명확화 필요

### 네이밍 오타
- `anlaysis` (analysis 오타) — 라우트 경로에 포함됨
- `widthdrawalStrategy` (withdrawal 오타)
- `investment-daignosis-planning` (diagnosis 오타)
- `contract-management` vs `ContactManagement` 불일치

### 사용자 흐름 개선 포인트
- 로그인 후 랜딩: 고객목록으로 이동 (대시보드 활용 검토 필요)
- 고객 정보 완성도 표시 기능 없음
- 외부 고객 접점 제한적 (본인인증/투자성향만)
- ROLE_ABROAD_GBI 역할의 UX 매우 제한적
