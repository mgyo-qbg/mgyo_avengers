## 사용 방법

- 사용자 테스트 요청 시 이 문서를 기본 참고 자료로 사용한다.
- 새로운 기능/플로우에 맞게 페르소나를 추가하거나 기존 페르소나를 수정한다.
- 수정 시 하단 **변경 이력** 섹션에 기록한다.
- 테스트 결과는 `user_test_results.xlsx`에 별도 기록한다.

## 기능 카테고리별 페르소나 선택 가이드

신규 기능 또는 플로우 테스트 시 아래 카테고리를 기준으로 페르소나를 선택한다.

| 기능 카테고리 | 권장 페르소나 | 이유 |
|-------------|-------------|------|
| 연금 정보 직접 입력 | A, C | 금융 정보 수준 높음, 직접 입력 선호 |
| 연금 정보 계산기 경유 | B, E | 금융 정보 낮음, 계산기 의존 |
| 퇴직연금 자산 확인 | A, E | DC 보유자 |
| 퇴직금 계산 | A, B, E | 근로소득자 다양한 케이스 |
| 자영업/비근로소득 입력 | C | 유일한 비근로소득자 |
| 기타 정기소득 입력 | C, E | 임대소득 등 기타소득 보유 |
| 복합 경력/이직자 케이스 | E | 이직 경력 보유, 가장 복잡 |
| 기본값 조정 UX | A, E | 기본값 수정 시나리오 포함 |
| 패시브 사용자 (기본값 유지) | B, C | 기본값 수정 없이 진행 |
| 조기 이탈 / 빈 상태 화면 | D | 서비스 조건 미충족 케이스 |
| 입력값 자동 채움 연동 | B, E | 이전 단계 입력값 다음 단계 자동 반영 |
| 전체 플로우 엔드투엔드 | E | 가장 복잡한 경로, 모든 분기 포함 |
